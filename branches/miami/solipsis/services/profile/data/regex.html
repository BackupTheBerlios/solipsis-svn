Filters on Solipsis

Default searching mode

Anyword entered in the searching fields will act as a key word and will be seek through neighbours' data. Filters are insensitive to case.

Advanced searching mode (files only, coming soon)

The keyword entered is not searched through, it is matched exactly. This means that "photo" will not give you any result for a picture named "my best photo ever.jpg", not even "photo.jpg". For those, you will have to use wildchars...

The character '*' is available as a wildchar (a joker) to ease and extend filtering results.

For instance the filter "*.jpg" matching anyfile with the extension".jpg". And "*forest*.jpg" would match any file containing "forest" in its title and with the extension ".jpg"

Expert searching mode (files only, coming soon)

This mode uses regular expressions to perform the search. It uses a set of wildchars to describe the format of the searched value, or the number of its occurences:
. ^ $ * + ? { [ ] \ | ( )

It also provides a set of expression describing the expected format:
\d \D \s \S \w \W

A very good HOWTO has been written by A.M. Kuchling and explains it all.
